# Маршруты, Сборка

Настольное приложение для обработки маршрутных XLS файлов.

---

## Требования

- Python 3.10 или выше
- Windows 10/11 или Linux

---

## Установка зависимостей

```bash
pip install PyQt6 xlrd xlwt
```

---

## Запуск

```bash
python app.py
```

При необработанных ошибках подробности записываются в файл **crash.log** в папке данных приложения (`%APPDATA%\ExcelRouteManager\` в Windows или `~/.config/ExcelRouteManager/` в Linux).

---

Linux/macOS — `~/.config/ExcelRouteManager/store.json`.

## Функционал

### Стартовая страница (Дашборд)
- **Обработать файлы** — переход к загрузке и обработке XLS
- **Последние (основной)** / **Последние (довоз)** — открыть последние сохранённые маршруты
- **Этикетки** — переход к созданию этикеток по шаблонам

### Обработка файлов
1. **Тип файла** — «Основной» или «Увеличение (Довоз)»
2. **Файлы ШК и СД** — отдельные списки; пропуск служебных строк (шапка/подвал) определяется автоматически
3. **Папка сохранения** (по умолчанию — Рабочий стол)
4. **Обработать** → при новых названиях продуктов открывается диалог: «Новый продукт» (добавить в справочник) или «Дубликат: …» (записать как вариант существующего — при следующих запусках определится автоматически)

### Предпросмотр — Общие маршруты
- Таблица всех маршрутов с продуктами; ширина столбцов подстраивается под содержимое (прочитать весь текст)
- **Ctrl + колёсико мыши** над таблицей — увеличить/уменьшить размер текста
- **Поиск** по адресу/номеру маршрута
- **Фильтрация** по продукту (компактный режим / маршруты полностью)
- **Редактирование** номера маршрута (двойной клик)
- **Исключение** маршрутов из итогового файла
- **Валидация** — проверка номеров маршрутов перед созданием файла
- Создание файла «Маршруты общие ДД.ММ.ГГГГ» и **этикеток XLS** (папка «Этикетки на ДД.ММ.ГГГГ»)

### Предпросмотр — По отделам
- Таблица маршрутов по отделам/подотделам; ширина столбцов подстраивается под содержимое
- **Ctrl + колёсико мыши** над таблицей — изменить размер текста
- Создание файлов по отделам и **этикеток по шаблонам**

### Этикетки
- Создание XLS по шаблонам продуктов (маршрут, дом, количество)
- **Настройки этикеток** — по отделам/подотделам: печать да/нет, шаблон XLS для каждого продукта
- Режимы: обычный, чищенка (≤5 кг на этикетку), сыпучка (два файла: до 4 кг / от 4 кг)

### Справочник продуктов (меню Справочники)
- **Левый список** — новые/непривязанные продукты; **правый** — привязанные к отделам (канонические названия)
- **Связка**: выбрать слева и справа → «Связать» или **перетащить** элемент слева на правый список (вариант → канонический)
- При следующем парсинге файлов вариант подставляется как каноническое название (удобно при опечатках и разных написаниях)
- Таблица алиасов внизу, кнопка ✕ — удалить связку
- При закрытии: если есть непривязанные продукты — предложение открыть «Отделы и продукты»

### Отделы и продукты (меню Справочники)
- Создание отделов и подотделов (один уровень вложенности)
- Привязка продуктов к отделам/подотделам
- При создании подотдела: автоматически «Печатать этикетки» и режим по имени (чищенка/сыпучка)

### Шаблоны (меню Справочники)
- 2 настраиваемых шаблона для файлов по отделам
- Выбор столбцов из списка: номер маршрута, адрес, продукт, ед. изм., количество, шт
- Изменение порядка столбцов перетаскиванием
- Привязка шаблона к отделу

### Настройки Шт (меню Настройки)
- Включение/выключение отображения в штуках для каждого продукта
- Округление **ШК** и **СД** по отдельности (для маршрутов из файлов ШК/СД)
- Все настройки сохраняются в JSON

### Горячие клавиши
- **Ctrl+O** — Обработка файлов (загрузка XLS)
- **Ctrl+L** — Этикетки
- **Ctrl+D** — Отделы и продукты
- **Ctrl+T** — Шаблоны
- **Ctrl+P** — Продукты
- **Escape** — Закрыть модальное окно или панель редактирования

## Логика расчёта штук

**Округление в большую сторону:**
- K = 2.7, pcsPerUnit = 1.5
- 2.7 / 1.5 = 1 шт (целая часть)
- остаток = 2.7 % 1.5 = 1.2 → 1.2 > 0.5 → +1 шт
- Итого: **2 шт**

**Округление в меньшую сторону:**
- K = 3.3, pcsPerUnit = 1.5
- 3.3 / 1.5 = 2 шт (целая часть)
- остаток = 3.3 % 1.5 = 0.3 → 0.3 < 0.5 → +0 шт
- Итого: **2 шт**

## Формат создаваемых файлов

### Общие маршруты
- Имя: `Маршруты общие ДД.ММ.ГГГГ ОС.xls` (ОС = Основной, УВ = Увеличение)
- Один лист на каждый маршрут (имя листа = номер маршрута)
- Строка 1: дата + тип файла
- Строка 2: заголовки (Номер маршрута, Адрес, Ед. изм., Количество, Шт)
- Далее: строки продуктов

### По отделам
- Имя: `Маршруты [название отдела] ДД.ММ.ГГГГ ОС.xls`
- Один лист на файл (имя листа = название отдела/подотдела)
- Маршруты отсортированы по возрастанию номера
- Строка 1: заголовок
- Далее: маршруты с продуктами

## Известные ограничения

- Поддерживается только формат XLS (Excel 97–2003). XLSX не поддерживается.
- Этикетки через Excel COM доступны только в Windows.
- Резервные копии store.json хранятся в количестве до 2 штук (store.json.bak1, store.json.bak2).

---

## Устранение неполадок

**Путь к логам:** при необработанных ошибках подробности записываются в `crash.log` в папке данных приложения:
- Windows: `%APPDATA%\ExcelRouteManager\crash.log`
- Linux/macOS: `~/.config/ExcelRouteManager/crash.log`

**Типичные ошибки:**
- «Не удалось загрузить store.json» — файл повреждён. Используйте «Настройки → Восстановить данные из резервной копии».
- Ошибки парсинга XLS — проверьте, что файл в формате Excel 97–2003 (.xls), а не .xlsx.
