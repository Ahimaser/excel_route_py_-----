# Варианты реализации: Справочник продуктов и добавление новых продуктов

## Текущее состояние

- **Справочник продуктов**: два списка (Без отдела | Привязанные), таблица алиасов (вариант → каноническое)
- **Новые продукты**: таблица с комбобоксом «Новый продукт» / «Дубликат: X»
- **Блокировка**: только на странице «Маршруты по отделам» — при непривязанных продуктах показывается заглушка

---

## 1. Отображение канонических названий в скобках

**Цель**: убрать отдельную колонку «Каноническое название», показывать его в строке продукта.

### Вариант 1A — Справочник продуктов (списки)

**Было**: `Морковь (кг)`  
**Стало**: `Морковь (Морковь свежая)` — для алиасов показывать вариант и в скобках каноническое.

```
Без отдела:
  Морковь (Морковь свежая)   ← алиас
  Картофель мытый (кг)        ← новый, без канонического

Привязанные:
  Морковь свежая (кг)        ← канонический, в скобках единица
```

### Вариант 1B — Таблица алиасов

**Было**: две колонки [Вариант | → Каноническое | ✕]  
**Стало**: одна колонка [Вариант (каноническое) | ✕]

```
Морковь (Морковь свежая)                    [✕]
Картофель мытый (Картофель)                 [✕]
```

### Вариант 1C — Диалог новых продуктов

**Было**: колонка «Название из файла» + «Действие» (комбобокс)  
**Стало**: в колонке «Название» показывать `Вариант из файла (→ каноническое)` при выборе «Дубликат».

```
Название из файла              Действие
Морковь (кг)                   [Новый продукт ▼]
Картофель мытый (кг)           [Дубликат: Картофель ▼]  → отображается как "Картофель мытый (→ Картофель)"
```

## 2. Альтернативные варианты окна «Справочник продуктов»

### Вариант 2A — Единая таблица с вкладками

Одна таблица, вкладки: «Все», «Без отдела», «По отделам».

| Название | Отдел | Ед. изм. | Действия |
|----------|-------|-----------|----------|
| Морковь (Морковь свежая) | — | кг | [В отдел] [✕] |
| Картофель | Овощи | кг | [✕] |

- Фильтр по отделу и поиск сверху
- Кнопка «В отдел» открывает выбор отдела для непривязанных
- Алиасы отображаются как «вариант (каноническое)» в одной колонке

### Вариант 2B — Дерево + панель деталей

Слева — дерево: Отделы → Подотделы → Продукты. Справа — «Без отдела» (список).

При выборе продукта справа — панель: каноническое, единица, кнопки «Связать с…», «В отдел».

### Вариант 2C — Карточки

Сетка карточек продуктов. Каждая карточка: название (каноническое в скобках), отдел, единица, бейджи «новый» / «алиас». Клик — редактирование.

## 3. Альтернативы диалога «Новые продукты»

### Вариант 3A — Inline в таблице предпросмотра

После обработки файлов — не отдельный диалог, а строка под таблицей:

```
⚠ 3 новых названия: [Морковь] [Картофель мытый] [Лук репчатый]
   Для каждого: [Новый ▼] [Дубликат: ... ▼]  [Применить]
```

Пользователь выбирает действие прямо на странице предпросмотра.

### Вариант 3B — Пошаговый мастер

Шаг 1: список новых названий.  
Шаг 2: для каждого — выбор «Новый» / «Дубликат» с автоподсказкой похожих.  
Шаг 3: подтверждение и привязка к отделам (опционально).

### Вариант 3C — Быстрые действия

Таблица: название | [Новый] [Дубликат ▼] | при выборе «Дубликат» — выпадающий список канонических с поиском.

## 4. Блокировка создания файлов до определения продуктов

**Цель**: не давать создавать файлы, пока все новые продукты не определены или не добавлены в отделы.

### Точки блокировки

1. **Кнопка «Обработать»** (home_page) — после парсинга, если есть `new_items` и пользователь не завершил диалог новых продуктов
2. **Кнопка «Создать файл «Общие маршруты»»** (preview_general_page)
3. **Кнопка «Создать файлы для всех отделов»** (preview_dept_page)
4. **Кнопка «Сохранить»** (сохранение маршрутов)
5. **Страница «Маршруты по отделам»** — уже блокируется при непривязанных продуктах

### Реализация

```python
def has_undefined_products(app_state) -> list[str]:
    """Продукты из маршрутов без отдела и не являющиеся алиасами."""
    routes = app_state.get("filteredRoutes", [])
    products = data_store.get_ref("products") or []
    aliases = data_store.get_aliases()
    assigned = {p["name"]: p.get("deptKey") for p in products if p.get("name")}
    undefined = []
    for r in routes:
        if r.get("excluded"):
            continue
        for p in r.get("products", []):
            name = p.get("name")
            canonical = aliases.get(name, name)
            if not assigned.get(canonical) and name not in assigned:
                undefined.append(name)
    return sorted(set(undefined))
```

### Вариант 4A — Баннер + отключение кнопок

Вверху страницы предпросмотра:

```
⚠ 3 продукта без отдела: Морковь, Картофель, Лук.
   [Открыть Отделы и продукты] [Справочник продуктов]
```

Кнопки «Создать файл», «Сохранить» — disabled, tooltip: «Сначала привяжите все продукты к отделам».

### Вариант 4B — Модальное предупреждение

При нажатии «Создать файл» — проверка. Если есть непривязанные:

```
Нельзя создать файлы: 3 продукта не привязаны к отделам.
[Открыть Отделы и продукты] [Отмена]
```

### Вариант 4C — Пошаговый flow

После обработки файлов:
1. Если есть новые — диалог «Новые продукты» (обязательно)
2. Переход на предпросмотр
3. Если есть непривязанные — баннер + disabled кнопки
4. Пользователь открывает «Отделы и продукты», привязывает
5. После привязки всех — кнопки включаются

## 5. Рекомендуемый набор изменений

| Компонент | Рекомендация |
|-----------|--------------|
| Канонические в скобках | Вариант 1A + 1B — везде показывать «вариант (каноническое)» |
| Справочник продуктов | Вариант 2A — единая таблица с вкладками, проще навигация |
| Новые продукты | Оставить текущий диалог, добавить отображение 1C |
| Блокировка файлов | Вариант 4A — баннер + disabled кнопок на preview_general_page и preview_dept_page |

## 6. Схема потока (наглядно)

```
[Загрузка XLS] → [Парсинг] → Есть новые названия?
                              ├─ Нет → [Предпросмотр]
                              └─ Да → [Диалог «Новые продукты»] → [Предпросмотр]
                                                                       │
                                    Есть непривязанные? ←───────────────┘
                                    ├─ Нет → Кнопки «Создать файл» активны
                                    └─ Да → Баннер + кнопки disabled
                                            [Открыть Отделы и продукты]
                                                    │
                                            Пользователь привязывает
                                                    │
                                            Обновление → кнопки активны
```
